🚦 FASTag Fraud Detection

A Cross-Modal Visual Verification System built with Classical CV

Engineered by B-Team

📦 1. Data Processing (src/01_explore_dataset.py)

Our pipeline targets structural taxonomy rather than standard objects. We isolated and categorized thousands of top-down toll plaza images into three distinct classes: f1 (Car), f4 (Bus), and f5 (Truck). Before extraction, every single ROI is forced into a standard grayscale 64x128 matrix to normalize aspect ratios.

⬇

📐 2. Structural Fingerprinting (src/02_hog_extraction.py)

Raw pixels are highly susceptible to lighting variance and color noise. We utilize Histogram of Oriented Gradients (HOG) to calculate first-order finite difference approximations (Gx, Gy) of the image derivative.

Parameters: 9 orientation bins, 8x8 pixels per cell, and 16x16 block normalization (L2-Hys).
Result: Each vehicle is compressed into a robust, 3,780-length 1D mathematical feature vector representing pure shape and edge geometry.
⬇

🧠 3. Maximum-Margin Classification (src/03_train_svm.py)

The 3,780-length feature matrices are passed through a StandardScaler to ensure zero-mean unit-variance, then fed into a Linear Support Vector Machine (LinearSVC). Because our dataset has a severe 33:1 imbalance (Cars vs. Buses), we applied class_weight='balanced' to heavily penalize misclassifying the minority classes.

Math: The SVM maps the HOG vectors into high-dimensional space to find the optimal hyperplane w·x + b = 0 that maximizes the margin between classes.
⬇

🔀 4. Cross-Modal Verifier (app.py & src/07)

The core logic of our system is a cross-modal verification layer. During inference, we run a softmax() function over the SVM's decision_function to generate a 0-100% confidence probability.

If the physical visual prediction (e.g., "Truck") contradicts the declared RFID FASTag class (e.g., "Car" tag to pay less), the system instantly calculates a Fraud Score and flags the vehicle for auditing.
