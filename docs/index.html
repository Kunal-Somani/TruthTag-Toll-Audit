Vehicle Classification (Car vs Truck)

This project classifies vehicles captured by a fixed toll camera into two classes: CAR or TRUCK. The approach avoids deep learning and instead uses classical computer vision features based on image gradients (edges), followed by a Support Vector Machine (SVM) classifier.

Processing Pipeline

  • Compute horizontal and vertical gradients (edge detection).
  • Convert gradients into edge strength (magnitude) and edge direction (orientation).
  • Normalize features to reduce the effect of lighting changes.
  • Use an SVM to classify the feature vector as CAR or TRUCK.

1) Edge Finder (Gradient Calculation)

Goal: Detect where the image intensity changes sharply. These changes correspond to object boundaries such as the sides and roofline of a vehicle.

Equations:

$$G_x = \text{PixelRight} - \text{PixelLeft}$$

$$G_y = \text{PixelUp} - \text{PixelDown}$$

Interpretation: For each pixel, compare opposite neighbors. A large $|G_x|$ indicates a strong vertical edge (e.g., the side of a truck). A large $|G_y|$ indicates a strong horizontal edge (e.g., roofline).

2) Edge Strength & Direction (Magnitude & Orientation)

Goal: Convert the raw gradients into two meaningful descriptors: how strong the edge is, and which direction it points.

Edge Strength (Magnitude):

$$m = \sqrt{G_x^2 + G_y^2}$$

Edge Direction (Orientation):

$$\theta = \arctan\left(\frac{G_y}{G_x}\right)$$

Practical note: Trucks tend to produce many strong, near-vertical edges (box-like geometry). Cars tend to produce more curved edges and less dominant vertical structure.

3) Lighting Robustness (Normalization)

Goal: Reduce sensitivity to global brightness changes caused by sun, clouds, shadows, or time of day.

Equation:

$$v_{norm} = \frac{v}{\sqrt{||v||^2}}$$

Interpretation: Divide the feature vector by its overall energy (magnitude). This keeps the relative pattern of features but removes scaling due to lighting.

Example:

  • Sunny: [100, 200, 100]
  • Cloudy: [10, 20, 10]
  • After normalization: both ≈ [0.4, 0.8, 0.4]

4) Decision Maker (Support Vector Machine – SVM)

Goal: Learn a separating boundary between the two classes using the extracted features.

Decision function:

$$f(x) = w \cdot x + b$$

Where: $x$ is the feature vector extracted from the image, $w$ is the learned weight vector, and $b$ is the bias term.

Classification rule:

  • If $f(x) > 0$ → TRUCK
  • If $f(x) < 0$ → CAR

The SVM learns $w$ and $b$ from labeled training examples so that the margin between classes is maximized.
